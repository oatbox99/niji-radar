# niji_gunma — ニジマス釣行判断システム（群馬・河川C&R / 特別釣場）

週末遠征の「無駄足」を減らすための意思決定ツール。**自分の釣果を含まず公開データのみで作る設計のため公開可能**（無料公開の手順は下記）。エビデンス（群馬県水試・EPA・USGS・査読論文 等）に基づき、ニジマスの釣りやすさと、キャッチ&リリース（C&R）で魚が生き残るかを推定する。

対応区間は**5つ**：上野村ハコスチ冬季C&R（`verified`）／利根川前橋・阪東子持・渡良瀬桐生・神流川鬼石（`参考`）。判定単位は「河川」でなく「区間(reach)」。

## エビデンスに基づく判断軸

- **水温が第一決定因子**（気温ではない）。摂餌至適 **10–16℃**、20℃超で摂餌停止、24℃接近で生息不適。
- **C&R死亡率は水温連動**。慣用20℃より低い**16℃から**上昇 → C&R区間は「釣れるか」以前に「離した魚が生きるか」で判定を止める（水温≥20℃→見送り）。
- **濁りはU字応答**：笹濁り＝最好条件、クリア＝やや渋い、泥濁り＝竿NG。
- **放流後経過日数**が管理C&R区間の最大ドライバー（0–3日＝荒食い）。
- **月齢/ソルナーは査読で予測力ゼロ → 使わない**（過剰主張回避）。

## データの現実（実装方針）

| データ | 状況 | 実装方針 |
|---|---|---|
| 日照+気温（AMeDAS） | ✅ クリーンJSON API | 気温=実測、日照=推計。日次に集計（mean/max/min） |
| **水温（1時間毎）** | ❌ ライブ源が存在しない | 気温からの**プロキシ**（未較正・現場報告優先） |
| 水位（観測点別） | ✅ カテゴリ取得可（一部区間） | Yahoo川ミラー。渡良瀬は観測点ID未確認→欠測表示 |
| 濁り/体感/水温/釣果/放流 | ⚠️ 区間による | 釣況ブログ・漁協 → Gemini構造化 |
| ダム放流 | ✅ 下久保/利根5基 | 八ッ場/草木はID未確認→「未確認」表示 |

**設計原則**
- **精度と網羅の両立**：全区間を網羅しつつ、公式ソース実在確認済み(`verified`)の区間だけが確信GOを出す。`参考`区間は物理データ＋注意書きに留める（確信GOを出さない）。
- **現場観測 > モデル**：新鮮なブログの体感報告があれば、TSI推定より優先。
- **危険・保全は最優先**：泥濁り・増水・ダム放流・高水温C&Rは、TSIが良くても見送り。
- TSIは公開データからの**推定**（自分の釣果では較正しない）。UIは常に信頼度・鮮度・出所を併記。
- 罠：同じ神流川でも上野村＝自然流量／鬼石＝下久保ダム放流支配で真逆（区間IDで紐付け）。AMeDAS上野(53112)は三重県（群馬は神流42396）。

## セットアップ

```bash
pip install -r requirements.txt
cp .env.example .env                    # GEMINI_API_KEY を記入（semantic用）
python -m scripts.seed_demo             # デモデータ投入（UI即体験）
streamlit run app/dashboard.py          # ダッシュボード
python -m scripts.run_ingest            # ライブ収集（--no-semantic でキー不要）
python -m scripts.daily_update out.html # 日次更新一括: 収集→判定台帳→静的HP
python -m ruff check --no-cache src app scripts  # lint
python -m pytest -q                     # test
```

## 無料公開（HP化）

自分の釣果を使わない設計＝個人情報が無いので公開に適する:

1. **静的HTMLスナップショット（最速・無料）** — `python -m scripts.daily_update out.html --full` で自己完結HTMLを書き出し、GitHub Pages / Netlify Drop / claude.ai Artifact に置くだけ。サーバ不要・APIキー不要（出力に残らない）。
2. **GitHub Pages + Actions（無料・自動更新）** — `.github/workflows/publish.yml` が日次で収集→静的HP→Pages配信（通年 07:30 JST）。`GEMINI_API_KEY` はActionのSecretに置き、生成HTMLには残らない。初回はリポ同梱の `data/niji_seed.db` から履歴を引き継ぐ。
3. **Streamlit Community Cloud（無料・双方向）** — 公開GitHubリポから常時稼働。

## 正直な限界

- **精度の天井は公開データの質で決まる**。水温プロキシは未較正で、ブログの水温記載で補正するまで過信しない。
- `参考`区間は公式ソースの実在/安定確認が済むまで確信GOを出さない（年度別URL変動・SSL失効・観測点ID未確認）。
- 八ッ場/草木のダムID、渡良瀬の水位観測点IDは未確認。冬期C&R区間は毎秋に再確認が必要。
- 引き際の時間窓・風・前線の影響はエビデンスが弱く（weak/anecdotal）、運用実績で較正する。
- semantic（Gemini実呼び出し）は要 `GEMINI_API_KEY`。非公式ツール。正確な期間・ルールは各漁協で確認を。
